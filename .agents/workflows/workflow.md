---
description: 4단계 바이브 코딩 워크플로
---

우리가 안티그래비티에서 프로젝트를 진행할 때 적용할 **[4단계 바이브 코딩 워크플로]**를 정의할게. 앞으로 모든 작업은 이 흐름에 맞춰서 진행해 줘.

1단계: [기획 및 확인]

내가 새로운 기능이나 수정을 요청하면, 코드를 먼저 짜지 말고 **'구현 방향성과 어떤 부분을 수정할 것인지'**를 사람의 말로 먼저 요약해서 나에게 설명해 줘. 내가 '진행해줘'라고 하면 그때 코드를 작성한다.

2단계: [점진적 코드 반영]

한 번에 여러 기능을 수정하지 않고, 딱 하나의 기능만 집중해서 반영한다.

전체 코드를 다 갈아엎지 말고, 수정이 필요한 부분만 정확히 짚어서 효율적으로 코드를 업데이트해 줘.

3단계: [실시간 테스트 및 피드백]

코드가 반영되면 나는 안티그래비티 미리보기 창에서 테스트를 할 거야.

만약 버그가 있거나 디자인이 어색하면 내가 피드백을 줄 테니, 넌 기존 기능을 망가뜨리지 않는 선에서 디버깅을 진행해 줘.

4단계: [체크포인트 완료 및 세이브]

하나의 기능이 완벽하게 작동하면, 넌 지금까지 구현된 핵심 기능 목록을 가볍게 브리핑해 주며 '이번 단계 완료! 다음엔 어떤 기능을 만들까요?'라고 물어봐 줘. (이 단계가 우리의 세이브포인트가 될 거야.)

이 워크플로를 확실히 이해했다면, **'워크플로 접수 완료!'**라고 말하고, 우리가 만들 'AI 아카이빙 홈페이지'의 **[1단계: 전체적인 레이아웃 기획(헤더, 입력 폼, 카드 리스트 배치 구상)]**을 먼저 제안해 줘. 코드는 아직 짜지 말고 기획안부터 보여줘!
